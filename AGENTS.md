## 3. Конвенции кода (важно соблюдать!)

- **Порядок в `<head>`:** favicon первым → title/description → canonical `https://kadry-pro.ru/<page>` → OG-теги (og:image logo.png 1200×630, og:locale ru_RU, og:site_name «Кадры-Про») → Schema.org JSON-LD → `style.css?v=3` (cache-busting!) → сниппет Метрики 108237323 + noscript.
- **Декоративный фон:** инлайн-блоки `.bg-decor` УДАЛЕНЫ со всех страниц. Теперь в body стоит `<div id="bg-decor"></div>`, а фон вставляет функция `initBgDecor()` из `script.js` (вызывается первой строкой в DOMContentLoaded). Не возвращать инлайн-фон!
- **Навигация** (одинакова на всех страницах): Главная / Услуги и цены / Подбор кадров / Обо мне / Трудовые споры / Кейсы / Шаблоны / Контакты + кнопка «Оставить заявку». Пункт текущей страницы с `class="active"`.
- **Футер** (одинаковый на 15 страницах; нет на bundle-download и thank-you): бренд с ценой «от 14 000 ₽/мес», колонка Услуги со ссылками-стрелками («Штрафы ГИТ 2026 →» и т.п.), Компания, Контакты; нижняя строка «© 2026 Мария Перепечай».
- **Тема:** `--bg-primary: #0a0a0a`, карточки `#1a1a1a`, акцент `--accent: #4ade80` / `--accent-dark: #22c55e`, шрифты Inter + JetBrains Mono, контейнер 1200px.
- **script.js:** прелоадер, nav.scrolled, гамбургер-меню, плавные якоря, IntersectionObserver `.anim`, счётчики `data-count`, калькулятор (тарифы в `getTierPrice()`, налог 30,2%, рабочее место 5 000 ₽), табы + `?tab=`, FAQ-аккордеон, маска телефона, валидация форм → Formspree, ripple, параллакс, cookie-баннер, фильтр кейсов.

## 5. Рабочий процесс (как сюда попадают правки)

- Пользователь работает с внешними AI-агентами (арена), те отдают результат **только md-дампами** в `agent_result/`. Задача здешнего агента — переносить правки из дампов в проект.
- Внешние агенты работают со старой копией репо — их диффы надо накладывать на ТЕКУЩЕЕ состояние, сверяя контекст.
- Перед массовыми правками — бэкап изменяемых файлов (конвенция чата 1).
- После правок — верификация: grep на старые значения, `node --check script.js`, проверка структуры (один head/body/html на страницу), валидность sitemap.xml.
- Временные скрипты удалять после работы.
- Пользователь проверяет сайт визуально сам, в конце всей работы («всё проверим, но позже, когда всё допилим»).
